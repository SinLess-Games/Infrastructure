# Incident Response Documentation and Resources

## What This Page Is For

This page is a curated reference for incident response, incident management, on-call operations, postmortems, and operational recovery.

It is meant to make it easier to find the right documentation quickly when working on:

- incident response processes
- on-call and escalation design
- runbooks and playbooks
- status pages and stakeholder communication
- postmortems and follow-up work
- incident metrics and operational improvement

## Useful Topics To Study

If you are building or improving incident response, focus on:

- incident severity definitions
- incident roles and command structure
- escalation policies
- stakeholder communication
- status pages
- runbooks and playbooks
- evidence capture
- timelines and incident logs
- postmortems
- metrics such as MTTR, MTTD, and incident volume
- follow-up ownership and corrective actions

## Suggested Learning Order

If you are learning incident response from scratch, this order works well:

1. NIST SP 800-61 Rev. 3
2. CISA incident response playbooks
3. Google SRE incident management guide
4. OneUptime incident management docs
5. PagerDuty incident response docs
6. Atlassian incident management handbook
7. incident.io practical guide
8. incident metrics and postmortem resources

## Notes

A few practical habits make incident response resources more useful:

- prefer official docs first
- use standards and playbooks to shape process
- use platform docs to shape tooling and automation
- treat postmortems as part of incident response, not something separate
- keep communications, escalation, and learning loops as important as technical remediation
