# Terraform and Butane Ignition Generation Guide

## Overview

This Kubernetes deployment uses a three-stage infrastructure-as-code pipeline:

1. **Butane → Ignition**: Declarative cloud-init for Flatcar Linux
2. **Terraform**: VM provisioning on Proxmox with Ignition payloads
3. **Ansible**: Orchestrates Terraform and RKE2 bootstrap

## Butane Templates

Located in `Ansible/templates/kubernetes/`:

### `flatcar-control-plane.bu.j2`

- Creates `sinless777` user with SSH keys
- Loads kernel modules: `br_netfilter`, `overlay`, `nf_conntrack`
- Applies sysctl tuning for Kubernetes (ip_forward, bridge rules, conntrack)
- Sets hostname and NTP
- Configures RKE2 server config stub

### `flatcar-worker.bu.j2`

- Same user + kernel setup as control-plane
- RKE2 agent configuration (minimal; server IP injected at runtime)

## Ignition JSON Generation

**Tool**: `butane` (install via `brew install butane` or similar)

**Process**:
```bash
# Ansible renders Butane template with jinja2 vars
ansible-playbook deploy-kubernetes-dev.yaml

# Ansible then compiles Butane → Ignition JSON
butane --strict --pretty \
  --output .outputs/kubernetes/dev/ignition/k8s-dev-cp-01.ign \
  .outputs/kubernetes/dev/butane/k8s-dev-cp-01.bu.yaml
```

**Output**: Three Ignition files per cluster (one per node):
- `.outputs/kubernetes/dev/ignition/k8s-dev-cp-01.ign`
- `.outputs/kubernetes/dev/ignition/k8s-dev-wk-01.ign`
- `.outputs/kubernetes/dev/ignition/k8s-dev-wk-02.ign`

## Terraform Modules

Located in `Terraform/modules/`:

### `kubernetes-master/`

Variables:
- `nodes`: list of control-plane VM specs (name, IP, VMID, hostname, Ignition JSON path)
- `vlan_id`, `network_bridge`: VLAN 20 for dev
- `cpu_cores`, `memory_mb`, `disk_size`: resource allocation

Special: Uses `-fw_cfg` QEMU argument to pass Ignition JSON directly:
```hcl
args = "-fw_cfg name=opt/com.coreos/config,string=${local.ignition_json_escaped[count.index]}"
```

### `kubernetes-worker/`

Same structure as `kubernetes-master` but with fewer resources (worker-optimized).

## Generated Terraform

**File**: `Terraform/kubernetes-dev-cluster.tf` (auto-generated by Ansible)

Template: `Ansible/templates/kubernetes/kubernetes-cluster.tf.j2`

Contains:
- `module "kubernetes_dev_masters"` - Instantiates master module for 1 control-plane
- `module "kubernetes_dev_workers"` - Instantiates worker module for 2 workers
- Output blocks for Ansible inventory generation

**Do NOT edit manually**—Ansible re-generates on each playbook run.

## Variable Flow

```
Ansible group_vars
  ↓
Butane template → .bu.yaml
  ↓
butane CLI → .ign JSON
  ↓
Terraform template → kubernetes-dev-cluster.tf
  ↓
Terraform file() function → reads .ign JSON and injects into VMs
  ↓
Proxmox VMs boot with Flatcar + Ignition config
```

## Idempotency

- **Butane**: Re-rendering template is safe; only changes on var updates
- **Terraform**: `state list` shows applied VMs; re-running plan detects drifts
- **RKE2**: Idempotent installation; systemd unit restart-on-failure

## Debugging

### Ignition Syntax Errors

```bash
# Validate Butane YAML before compilation
butane --strict --data-dir . Ansible/templates/kubernetes/flatcar-control-plane.bu.j2

# Check compiled JSON (pretty-print)
cat .outputs/kubernetes/dev/ignition/k8s-dev-cp-01.ign | jq .
```

### Terraform Plan Issues

```bash
# Ensure Ignition files are readable and not empty
ls -lh .outputs/kubernetes/dev/ignition/*.ign

# Show Terraform plan with Ignition contents (first 200 chars)
terraform plan -target=module.kubernetes_dev_masters | grep -A 5 ignition_json
```

### Flatcar Boot Logs

```bash
# SSH to newly booted node
ssh sinless777@10.10.20.11

# Check if Ignition was applied
sudo systemctl status afterburn
sudo journalctl -u ignition-diskful --no-pager

# Verify kernel modules loaded
lsmod | grep -E 'br_netfilter|nf_conntrack'

# Check final sysctl
cat /etc/sysctl.d/90-kubernetes.conf
```

## Testing Changes

### Render without Apply

```bash
task ansible:k8s:dev:plan
```

Stops after Terraform plan; does NOT create/modify VMs.

### Verify Ignition Contents

```bash
# After task ansible:k8s:dev:plan
cat .outputs/kubernetes/dev/butane/k8s-dev-cp-01.bu.yaml
cat .outputs/kubernetes/dev/ignition/k8s-dev-cp-01.ign | jq .
```

### Destroy and Redeploy

```bash
task ansible:k8s:dev:destroy  # Terraform destroy only
# Manual cleanup of .outputs/kubernetes/dev/ if desired
task ansible:k8s:dev         # Full redeploy
```

## Adding Custom Ignition Config

1. **Edit template**:
   ```yaml
   # Ansible/templates/kubernetes/flatcar-control-plane.bu.j2
   storage:
     files:
       - path: /etc/myapp/config.yaml
         contents:
           inline: |
             key: value
   ```

2. **Re-run playbook**:
   ```bash
   task ansible:k8s:dev:plan
   ```

3. **Review changes**:
   ```bash
   cat .outputs/kubernetes/dev/ignition/k8s-dev-cp-01.ign | jq . | grep -A 10 myapp
   ```

4. **Apply if correct**:
   ```bash
   task ansible:k8s:dev
   ```
